# Using Vite+, the Unified Toolchain for the Web

This project is using Vite+, a unified toolchain built on top of Vite, Rolldown,
Vitest, tsdown, Oxlint, Oxfmt, and Vite Task. Vite+ wraps runtime management,
package management, and frontend tooling in a single global CLI called `vp`.
Vite+ is distinct from Vite, and it invokes Vite through `vp dev` and
`vp build`. Run `vp help` to print a list of commands and `vp <command> --help`
for information about a specific command.

Docs are local at `node_modules/vite-plus/docs` or online at
https://viteplus.dev/guide/.

## Review Checklist

- [ ] Run `vp install` after pulling remote changes and before getting started.
- [ ] Run `vp check` and `vp test` to format, lint, type check and test changes.
- [ ] Check if there are `vite.config.ts` tasks or `package.json` scripts
      necessary for validation, run via `vp run <script>`.
- [ ] If setup, runtime, or package-manager behavior looks wrong, run
      `vp env doctor` and include its output when asking for help.

## Stack

- **TanStack Start** SSR framework ‚Äî file-based routing `src/routes/`, router
  `src/router.tsx`. SSR + TanStack Query integration wired in `src/router.tsx`
  via `setupRouterSsrQueryIntegration`; query context in
  `src/lib/query/context.ts`.
- **Bun** package manager (`bunfig.toml`, `devEngines`). Run `bun install`.
  `bunfig.toml` sets `install.peer = false`.
- **Nitro** server layer (nitro nightly, in vite plugin; externalizes
  `@sentry/*`).
- **Tailwind CSS v4** via `@tailwindcss/vite`; base CSS at `src/styles.css`.
- **React 19**, **TanStack Query**, **TanStack Form**, **TanStack Table**.
- **t3-env** validates env vars (`src/lib/env.ts`); client prefix is `PUBLIC_`.
- **API client**: `openapi-fetch` typed client in `src/lib/api/client.ts`, typed
  by generated OpenAPI types.

## Environment variables

Only two vars, both `PUBLIC_`-prefixed (client-exposed), validated by t3-env:

- `PUBLIC_API_URL` ‚Äî API base URL used by `fetchClient`.
- `PUBLIC_APP_TITLE` ‚Äî app title.

`vite.config.ts` `envPrefix` also allows `VITE_`, but t3-env only exposes
`PUBLIC_*`. `.env.example` is the source of truth for required keys.

## Path aliases

`@/` ‚Üí `./src/` (configured in `tsconfig.json` paths and `package.json`
imports). `no-relative-parent-imports` lint rule is `error` ‚Äî use `@/` for
cross-package imports.

## File conventions

- `src/routeTree.gen.ts` ‚Äî auto-generated by `tsr generate`, **never edit
  manually**. Lint/fmt ignore it.
- `src/routes/` ‚Äî TanStack Start file-based routes. `__root.tsx` is root layout,
  `index.tsx` is `/`.
- **kebab-case** filenames (e.g. `pagination-button.tsx`). Exceptions:
  `routeTree.gen.ts`, `__root.tsx`, `index.tsx`.
- `CLAUDE.md` is a symlink to `AGENTS.md` ‚Äî editing this file updates both.

## Generated API types (gotcha)

`src/lib/api/types.d.ts` is a large generated file (OpenAPI ‚Üí TypeScript via
`openapi-typescript`). `client.ts` imports it via relative `./types`.

The `generate-api-types` script in `package.json` writes to
`./src/api/types.d.ts` ‚Äî a **stale path** that does not match where `client.ts`
imports from. Before running `bun run generate-api-types`, fix the `-o` path to
`./src/lib/api/types.d.ts`, or the regeneration will not update the file
actually used. Lint/fmt ignore `src/api/types.d.ts` and `src/lib/api/types.d.ts`
via `vite.config.ts`.

`predev` runs `generate-api-types` automatically before `vp dev`.

## Components (shadcn / COSS UI)

- `src/components/ui/` ‚Äî reusable UI components. Check existing components first
  before creating new ones.
- shadcn registry ‚Äî primary source for new components (`shadcn` CLI,
  `https://ui.shadcn.com/r`). `components.json` configured: `new-york` style,
  `neutral` base color, `lucide` icons, CSS variables on, utils alias
  `@/lib/utils/style`.
- COSS UI ‚Äî `https://coss.com/ui/llms.txt` for component references, use only
  after checking shadcn. COSS registry configured in `components.json` as
  `@coss`.
- Prefer COSS components over shadcn by default. New components go in
  `src/components/ui/`.
- **Components MUST be modular and reusable.** Single responsibility, composable
  props, no page-specific logic. Compose small components together rather than
  building monolithic ones.
- **Never create components inside `src/routes/`.** Routes are for routing,
  layout, and data loading only ‚Äî they may use components but must not define
  them. Any UI used in a route belongs in `src/components/` (or
  `src/components/ui/` for generic primitives). Route files should stay thin.
- `cn` helper lives at `@/lib/utils/style` (clsx + tailwind-merge).

## Style

- **Functional programming only.** No classes. Use functions, function
  components, plain objects. Type helpers via interfaces/types, not class
  inheritance.
- **No comments.** Code should be self-documenting. No JSDoc, no inline
  comments, no file headers. **Avoid `useEffect`.** Use React events, TanStack
  Query, or React 19 patterns (`use`, Suspense) instead of side-effect hooks.
- `no-console` lint rule is `error` ‚Äî only `console.error`, `console.warn`,
  `console.info` allowed. No `console.log`.
- **Formatter (oxfmt) defaults** (see `vite.config.ts`): `semi: false`,
  `singleQuote: false` (double quotes), `tabWidth: 2`, `trailingComma: "all"`,
  `printWidth: 80`, `bracketSpacing: true`, `jsxSingleQuote: false`,
  `proseWrap: "always"`. No semicolons is intentional ‚Äî do not add them.
- Imports sorted: builtin ‚Üí external ‚Üí type-workspace ‚Üí value-workspace ‚Üí
  type-internal ‚Üí value-internal ‚Üí parent/sibling/index ‚Üí unknown.
  `no-relative-parent-imports` enforced.
- Use `import type { ... }` for type-only imports (separate-type-imports
  enforced).
- When importing icons from `lucide-react`, always append `Icon` to the name
  (e.g., `import { StarIcon } from 'lucide-react'`).

## React Doctor

Run `bunx react-doctor@latest --verbose` periodically to catch TanStack Start
route-order issues, query destructuring mistakes, hydration flickers, unstable
context values, and React 19 migration issues.

Common fixes observed in this repo:

- **Route property order**: put `loader` / `validateSearch` / `beforeLoad`
  before `head` so TanStack Router type inference works
  (`tanstack-start-route-property-order`).
- **TanStack Query**: destructure `useQuery()` / `useInfiniteQuery()` results,
  e.g. `const { data, isLoading } = useQuery(...)` instead of assigning the
  whole object (`query-destructure-result`).
- **Theme / hydration**: replace `useEffect`-driven theme state with
  `useSyncExternalStore` and React 19 `use` for context reads.
- **Context providers**: memoize the `value` object with `useMemo` to avoid
  re-rendering consumers every render.
- **Tooling note**: the Read tool can visually drop braces, `from`, and
  semicolons. Verify actual bytes with `xxd`/`hexdump` or use Node
  `fs.writeFileSync` replacements when bulk-rewriting files.
